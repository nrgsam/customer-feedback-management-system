# Customer Feedback & Complaint Management System

## Overview
This project is a **web-based Customer Feedback and Complaint Management System** developed as part of an **internship program**.  
The system is designed for **organizational use** and enables customers to submit **feedback, suggestions, and complaints**, while allowing administrators to manage and track submitted records efficiently.

The platform improves transparency by providing customers with a **tracking code** to follow the status of their complaints.

---

## Project Type
- Internship Project  
- Organizational / Enterprise-Oriented System  
- Web-Based Application  

---

## Key Features
- Submit customer feedback, suggestions, and complaints
- Generate a unique **tracking code** for each complaint
- Customers can check complaint status using the tracking code
- **Admin-only login system**
- Admin panel for reviewing, updating, and managing complaints
- Structured data storage using a relational database

---

## Technologies Used
- **HTML** – Page structure  
- **CSS** – Styling and layout  
- **JavaScript** – Client-side validation and interactions  
- **PHP** – Server-side logic  
- **MySQL** – Database management system  

---

## User Roles

### Customer
- Submit feedback, suggestions, or complaints
- Receive a tracking code after submission
- Track complaint status using the tracking code without creating an account

### Admin
- Secure login access
- View all submitted complaints
- Update complaint status
- Track and manage customer submissions

---

## How the System Works
1. A customer submits feedback or a complaint through a web form.
2. The system stores the information in a **MySQL database**.
3. A unique tracking code is generated and provided to the customer.
4. The customer can check the complaint status by entering the tracking code.
5. An admin logs in to the system to review and update complaint statuses.
